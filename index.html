Drupal Web Components

Native Web Components (Lit + TypeScript) for rendering Drupal JSON:API content

Portfolio demo · mock data

Content Listing

<drupal-content-search> + <drupal-content-list> wired together. The list fetches from a configurable JSON:API endpoint (mock data here). Type to filter client-side; the result count is announced to screen readers via aria-live.

Content Card (standalone)

<drupal-content-card> accepts a ContentItem object and renders with full Shadow DOM encapsulation. Slots let you inject arbitrary content.

Accessible Accordion

<ui-accordion> is built entirely from scratch — no third-party dependencies. It implements the WAI-ARIA Accordion pattern : correct aria-expanded, aria-controls, and keyboard navigation (↑↓ Home End).

The JSON:API module ships with Drupal 8.7+ core. It exposes every entity type as a standards-compliant REST API with consistent URLs, filtering, sparse fieldsets, and includes — no custom endpoint code required.

Web Components run in every modern browser without a framework runtime. For a Drupal-rendered page, that means zero JavaScript tax on the host theme — just drop in a <script type="module"> tag and use the element. They compose with any CMS, server-rendered or otherwise.

Set the endpoint attribute on <drupal-content-list> to your Drupal JSON:API URL and remove use-mock: 

<drupal-content-list
  endpoint="https://your-site.example/jsonapi/node/article"
  base-url="https://your-site.example"
></drupal-content-list>

You may need to configure CORS in services.yml to allow requests from your front-end origin. See the README for full details.

All components target WCAG 2.1 AA:

  • Semantic HTML with correct ARIA roles and states
  • Visible focus indicators (3 px accent ring)
  • Keyboard-navigable without a mouse
  • aria-live regions for async content announcements
  • prefers-reduced-motion respected on all transitions
  • Light/dark colour schemes via CSS custom properties
